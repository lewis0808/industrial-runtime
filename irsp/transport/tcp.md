# 传输 V3 — TCP / TLV（预留）

> **状态：预留，V1/V2 不实现。** 通过 `HELLO ... transport=tcp` 协商启用。

## 目标

为高吞吐 / 低延迟场景（大量 Tag、Stream、内网采集）提供裸 TCP 传输，
**IRP 语义与编码（resp1 或 msgpack）保持不变**。

## 帧定界

TCP 是无边界字节流，需自行定界。约定：每个 IRP 帧前加 **4 字节大端长度前缀**：

```
[ uint32 BE length ][ length 字节的编码层载荷 ]
```

载荷即编码层（resp1/msgpack）的一个顶层值。除定界外，语义层与编码层完全复用。

## 与 WebSocket 的差异

| 维度 | WebSocket(V1) | TCP(V3) |
|------|---------------|---------|
| 消息边界 | WS 帧自带 | 4 字节长度前缀 |
| 浏览器可直连 | 是 | 否 |
| 开销 | WS 帧头/掩码 | 最小 |
| 适用 | Web HMI / 通用客户端 | 高性能内网采集 / 服务间 |

## 约束

- 语义层（命令、错误、Topic Tree）零改动；客户端切换传输不改业务代码。
- TLS、鉴权等细节在落地 V3 时补全本文件。
